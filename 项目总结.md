# Editing V1 — 项目总结

> AI 流水线：医生讲课视频 → 语义分析 → 动效 overlay → 编辑器审核 → 渲染输出

更新日期：2026-03-24

---

## 这是什么

一个本地优先的 TypeScript + Python 单体仓库，完成以下工作：

1. 用 **Qwen3-ASR** 转录医生讲课 `.mp4`，生成字级时间戳
2. 通过 **4 阶段 LLM 链**（默认 Gemini 3.1 Pro）分析文稿，产出三层语义 `blueprint.json`
3. 将保留片段与原始音频做 **时间对齐**，生成 `timing_map.json`
4. 用 Remotion + FFmpeg **渲染** 动效图形 overlay（`overlay_layer.mp4`）+ 剪切后的原始视频（`source_direct_cut_video.mp4`）
5. 导出 **SRT 字幕**
6. 提供 **浏览器编辑器**（端口 3456）供人工审核和模板微调

最终在剪映 / DaVinci 中合成，属于本仓库边界之外。

## 核心数据模型

```
Blueprint（根节点）
└── BlueprintScene[]        （话题/意图边界 → overlay 模式）
    └── LogicSegment[]      （视角切换 → 一个模板块）
        └── BlueprintAtom[] （最小不可拆单元 → keep/discard + 时间戳）
```

每个 atom 关键字段：`text`、`status`（keep|discard）、`template_id`、`items[]`、`start_ms / end_ms`

## LLM 流水线（4 个阶段，位于 `src/analyze/`）

| 阶段 | 名称 | 输入 | 输出 |
|---|---|---|---|
| 0 | Qwen3-ASR | `.mp4` | `transcript.json`（字级时间戳）|
| 0.5 | Qwen3-强制对齐 | transcript + 音频 | `transcript_aligned_tokens.json` |
| 1 | Review | transcript + 可选 `.docx` | `reviewed_tokens.json` |
| 2 | Step1 | reviewed tokens | step1 atom 列表 |
| 3 | Take-Pass | step1 atoms + 音频块 | keep/discard 决策 |
| 4 | Step2 | 分组后的 atoms | 模板分配 → `blueprint.json` |

**默认 LLM：** `claude-sonnet-4-6`（Anthropic API）
**环境变量：** `ANTHROPIC_API_KEY`

## 完整 E2E 命令（API Key 模式）

```powershell
$dir = "P:/团队空间/.../xxx kp01"
$out = "$dir/out"
$mp4 = (Get-ChildItem "$dir/*.mp4" | Select-Object -First 1).FullName
$docx = (Get-ChildItem "$dir/*.docx" | Select-Object -First 1).FullName

# 1. 分析（转录 + 强制对齐 + 4 阶段 LLM）
npm run analyze -- --audio "$mp4" --script "$docx" -o "$out/blueprint.json" --transcribe-qwen --force-align-qwen

# 2. 时间图
npm run timing:direct -- --input "$mp4" -b "$out/blueprint.json" -o "$out/timing_map.json"

# 3. 渲染（overlay.mp4 + source_direct_cut_video.mp4）
npm run render -- -b "$out/blueprint.json" -t "$out/timing_map.json" --source-video "$mp4" -o "$out/overlay.mp4"

# 4. 字幕
npx tsx src/renderer/export-srt.ts -b "$out/blueprint.json" -t "$out/timing_map.json" -o "$out/subtitles.srt"

# 5. 编辑器（浏览器打开 http://localhost:3456）
npm run editor:serve -- "$out"
```

**断点续跑（已有 transcript，跳过转录和 review）：**
```powershell
npm run analyze -- --transcript "$out/transcript_aligned_tokens.json" --skip-review -o "$out/blueprint.json"
```

## 编辑器技术栈

- **前端：** React 19 + Vite（开发端口 5173，生产从 `dist/` 静态托管）
- **后端：** `server.ts` — Node.js HTTP，端口 3456
- **状态管理：** Zustand `blueprint-store.ts`
- **主要 API 路由：** `GET/PUT /api/blueprint`、`POST /api/still`、`POST /api/render`、`POST /api/finalize`

```powershell
npm run editor:build              # 构建前端 → dist/
npm run editor:serve -- <out目录>  # 启动服务
```

## 当前模式

**直出模式**：分析 → 时间图 → 渲染，全程命令行，不经过编辑器。编辑器（`app/` + `server.ts`）代码保留在仓库中备用，当前不使用。

## 待办事项

### 高优先级

1. **长视频 CLI 超时** — 1800+ 词在 CLI 模式下 step1 仍有风险（已调至 900s），API Key 模式稳定。

### 中优先级

2. **素材系统未建立** — 无 `asset_index.json`、无高频词素材预匹配、无偏好学习（`preferences.json`）。

3. **Blueprint schema 缺字段** — 无顶层 `discarded[]` 数组，无稳定的 `asset` / `background` 字段。

### 低优先级 / 待决策

4. **编辑器渲染队列** — 若将来启用编辑器，`POST /api/render` 是单任务异步，无进度反馈。

5. **局域网部署 + 多人并发** — 渲染队列、多实例支持均未开始。

6. **素材驱动 vs 语义结构方向** — 原计划「素材替换驱动」，现实现为「语义结构 + 模板编辑」，需决定是否补充素材系统作为第二阶段。
