# 知识视频 AI 内容引擎 · 工作系统待办全览（现状注释版）

更新日期：2026-03-14

## 标签说明

- `已完成`：当前仓库里已经有可用实现，且主路径能跑通。
- `部分完成`：已经有实现，但和原计划不完全一致，或者缺关键一环。
- `未开始`：当前仓库里基本看不到对应实现。
- `已偏航`：当前实现方向已经明显不同于原计划，建议你重新确认是否继续按现在路线推进。

---

## 一句话定义

`部分完成` 原始视频 -> AI三级语义分析 -> 编辑器 -> 渲染，这条主链路已经能本地跑通。

`部分完成` 系统边界大体符合“只做信息图形，不做人像抠图和最终合成”，但当前输出更像“带音频的信息图视频成片前体”，而不是严格意义上的纯 overlay 层。

`已偏航` 原计划强调“剪辑师主要做素材替换，系统输出 overlay 层”；当前项目更成熟的是“语义编辑 + 模板编辑 + keep/discard 调整”，素材系统并没有真正建起来。

---

## 二、三级语义标注体系

`已完成` 三层结构已经落地：`Scene -> LogicSegment -> Atom`。

`已完成` keep / discard、scene / logic boundary 的基本语义链已经存在。

`部分完成` 三级结构的稳定性还没有被多视频验证，最近还修过一次 `Step2 -> merge` 丢 logic block 的问题，说明结构正确性仍在收敛期。

## 三、完整工作流

### Step 1: 音频提取 + faster-whisper 转录

`已完成` 已集成 faster-whisper，输出字级时间戳。

### Step 2: LLM 三级语义分析

`部分完成` Step1/Step2 两步语义分析已存在，且已经加了 docx 参考清洗、硬清洗、语义去重。

`部分完成` 质量还没有稳定到“换视频也放心”的程度，仍需要多样本验证。

### Step 2.5: Words 后处理对齐

`已完成` 已有后处理对齐层，把 atom 文本重新对齐回 transcript words。

### Step 3: preferences.json 硬替换

`未开始` 当前没有真正的 `preferences.json` 飞轮，也没有自动按历史偏好替换素材。

### Step 4: 编辑器（人工微调）

`部分完成` 编辑器已有三层浏览、模板切换、items 编辑、废料恢复、still 预览、cut video 预览。

`部分完成` 但它还不是计划里的“素材驱动型编辑器”，更像“语义结构编辑器”。

### Step 5: Remotion 服务端渲染

`部分完成` 已经可以服务端渲染，并产出 `result.mp4`。

`部分完成` 当前没有真正的渲染队列，编辑器里的“渲染状态”也不是真实队列状态。

### Step 6: 剪辑师在剪映完成最终合成

`已完成` 这件事作为系统外部边界是成立的；仓库本身并不负责这一步。

## 四、 blueprint JSON 结构（三级）

`部分完成` 现有 schema 已经是三层结构，核心字段 `title / scenes / logic_segments / atoms / status` 都有。

`已偏航` 和原计划里的 JSON 仍有明显差异：
- 当前 schema 没有稳定的 `asset`
- 当前 schema 没有 `asset_source`
- 当前 schema 没有 `background`
- 当前 schema 没有顶层 `discarded` 数组
- 当前 schema 更偏向“逻辑段 + items”，而不是“atom 级素材配置”

`已偏航` 这意味着：如果后面要做素材偏好飞轮，需要先确认是继续沿用现在的 `items` 模型，还是回到计划里的 `atom.asset` 模型。

## 我对当前项目的一句话定位

当前项目最准确的描述不是“计划全部完成”，而是：

**一个已经跑通的三级语义视频处理样机 + 可用的中间编辑器。**

它最强的部分是：
- 端到端主链路
- 三层语义结构
- 中间预览和人工修正入口

它最弱、也最需要你决定下一步方向的部分是：
- 素材系统到底还做不做、按什么模型做
- preferences 飞轮要不要继续按原计划推进
- 当前架构是否继续沿用“先 cut_video，再 render”这条路线

如果你接下来要排计划，我建议优先把三件事定死：
1. 继续做“语义编辑器”，还是回到“素材编辑器”主轴
2. 是否保留当前 cut_video 架构，不再回头做原视频跳播
3. 素材 / preferences 系统是不是第二优先级，而不是继续打磨 prompt
