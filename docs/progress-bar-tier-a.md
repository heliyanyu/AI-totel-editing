# 进度条改进方案：档位 A（折叠式两级结构）

## 这个改动不是今天的主要工作

这是从"剪辑分层架构"讨论中分出来的一个**独立小改动**。改动小、不依赖任何新 LLM 调用、不依赖新数据流，但能单独跑出可见效果。记录在这里以便后续单独推进，不阻塞主线工作。

## 当前进度条是什么

实现：[src/remotion/overlays/TopProgressBar.tsx](../src/remotion/overlays/TopProgressBar.tsx)

形态：**水平一维 chip 序列**——把视频所有 scene 节点排成一行，顶部固定 96px 高度（占屏幕约 5%），按播放进度切换 active 状态（chip 变色）。

```
┌──────────────────────────────────────────────┐
│  ◉鱼油  ─  ○牛初乳  ─  ○灵芝  ─  ○免疫力  ...  │  ← 96px 高
├──────────────────────────────────────────────┤
│                                              │
│              内容主体区                       │
│                                              │
└──────────────────────────────────────────────┘
```

## 为什么要改

进度条本质上是剪辑分层架构里的 **L1 导航/结构层**——它告诉观众"现在讲到哪、整体多少内容"。这一层在中老年观众群里**特别重要**——这个用户群最容易在视频中"跟丢"，导航层让他们随时能定位自己在叙事中的位置。

但当前进度条只表达了**一维的 scene 序列**，丢失了 scene 内部的结构信息。一个常见痛点：观众讲到"鱼油的真相"这一节中段时，他知道"在第 1 个产品里"，但不知道"这个产品讲到第几小节、还剩多少"。结果就是：知道在哪个章节，但不知道章节内的进度。

参考行业里高质量医疗科普短视频的常见做法——画面顶部常驻一个**两级结构图**，一级是大章节（"6 种误区/3 个产品"），二级是当前章节内的小节展开。观众随时一眼看到全局结构 + 当前精确位置。

## 怎么改（档位 A）

**改进目标**：在不动渲染管线、不动 step1/step2 LLM 调用、不引入新数据的前提下，把进度条从"一维 chip 序列"升级为"两级折叠结构"。

### 视觉形态

- 一级节点：scene chip（当前形态保持不变）
- 二级节点：logic_block 子 chip，仅在**当前激活 scene 下方展开**，其他 scene 折叠
- 顶部空间从 96px 扩到约 180-200px（占屏幕约 10%）
- 二级节点用更小字号（约 26-28px，对比一级 40px）+ 更细描边
- 一级 chip 之间用横线连接（已有），二级 chip 用细竖线连接到所属一级 chip

```
┌──────────────────────────────────────────────┐
│  ◉鱼油 ── ○牛初乳 ── ○灵芝 ── ○免疫力 ...     │  ← 一级 ~96px
│   │                                          │
│   ├─ ●房颤+10%   ○反讽案例   ○出血叠加  ○建议  │  ← 二级 ~80px（仅当前展开）
└──────────────────────────────────────────────┘
```

### 数据来源

完全基于已有数据，**不需要新 LLM 调用**：

- 一级节点：[blueprint.json](../../) 的 `scenes[].title`（step2 已生成）
- 二级节点：`scenes[].logic_segments[].transition_type` 或 items 摘要（step2 已生成）
- 时间映射：`logic_segments[].atoms[].time` 区间（step1 已生成）

### 涉及的代码改动

1. **[src/remotion/overlays/TopProgressBar.tsx](../src/remotion/overlays/TopProgressBar.tsx)**
   - 加二级 chip 渲染逻辑
   - active scene 下方展开当前 logic_block 列表
   - 二级 chip 跟随当前 atom 时间高亮 active 项
   - 折叠状态切换的过渡动画（fade + height transition）

2. **[src/compose/visual-planner.ts](../src/compose/visual-planner.ts)**
   - `VisualSegmentPlan` / `VisualTopicNode` 增加 `subNodes: SubNode[]` 字段
   - 从 blueprint 提取 logic_block 数据填充

3. **[src/remotion/design-system.ts](../src/remotion/design-system.ts)**
   - `SAFE_AREA.top` 从 200 改到约 280-300（顶部内容区让出新增空间）
   - 内容区 `contentHeight` 同步缩减约 80-100px

4. **不动**的部分（重要）：
   - 不动 step1 / step2 LLM prompt
   - 不动渲染流水线（[run-batch.ps1](../run-batch.ps1)）
   - 不动剪映草稿生成
   - 不动资产匹配

## 验证方式

1. 拿 mianyili 03 这个案例（已跑过完整 pipeline）重新渲染 overlay.mp4
2. 跟当前版本并排对比观感
3. 重点看：
   - 顶部空间扩大后，内容主体区是否被挤压得不舒服
   - 二级 chip 切换是否跟得上语速（有的 logic_block 只有 4 秒）
   - 折叠/展开过渡是否流畅
   - 中老年观众试看反馈（剪辑师团队 6 人即可）

## 跟主线工作的关系

这是 L1 导航层的**最小可行改进**。完成后 L1 形态多样化（思维导图常驻、章节列表、阶梯进度等）属于档位 B/C，是档位 A 的扩展。

**主线（剪辑分层架构）该怎么走仍然按主线讨论结果决定**，这个文档只描述档位 A 这一个独立改动。两边可以并行，不阻塞。
